从零开始的VPS酒馆搭建（喂饭教程）

前言

利用VPS搭建酒馆有不少好处，比如稳定运行、性能与扩展性强、网络优化好（国外VPS）、集成其他服务（caddy），最重要的就是方便！方便！还是方便！不限制使用设备，有网就行（不用梯子🐶（国外VPS）），随时随地想用就用。不过缺点也很明显，首先就是搭建难度稍微大一丢丢，其次是需要一些钱（购买服务器、域名）。

那么，接下来本喂饭教程就从最最基本的服务器购买开始，破除技术壁垒，一步一步带领纯小白成功搭建，话不多说，马上开始！（多图警告）

服务器相关

服务器选择

最近也是找了一圈VPS，最后选择的是腾讯云的新加坡轻量应用服务器，原因就是国外平台对于小白来说不好操作，其次就是价格算是比较便宜（99一年，可原价续费一次），也不用梯子了，想要选择其他的也可以，不过本教程按腾讯轻量应用服务器为例。

购买链接：https://mc.tencent.com/o2Uhfxg9

打开此链接并下拉，可以看到箭头指的服务器，选择购买，我已经买过了就截不了图了，按教程安装系统即可（Ubuntu为例）。

ok，到了这一步之后，最最基础的服务器已经准备好了，接下来是安全配置。

服务器安全（可选，建议）

修改22端口

在云服务器（如VPS）上修改默认的 22端口（SSH服务端口）主要是出于安全加固的目的，这是服务器运维中的一项基础但关键的安全措施，具体如下：

输入命令sudo vim /etc/ssh/sshd_config修改文件，找到 #Port 22（默认是注释的），取消注释添加新的端口（例如 8765）。

修改完成之后按下esc键，输入:wq并回车完成修改。

实际操作过程中可能出现以下问题

直接按q退出，然后把vim的临时缓存文件删除就行了sudo rm -rf /etc/ssh/.sshd_config.swp

修改后使用命令重启ssh服务sudo systemctl restart sshd

放行防火墙，依次输出以下命令

回到控制台，进入防火墙，添加刚刚设置的端口

然后重新开一个终端使用新端口登录

如果成功登录（一定一定要确认新端口可以登录，不然就锁外面了），再次修改文件sudo vim /etc/ssh/sshd_config，这次将22端口注释掉。

然后reload ssh服务sudo systemctl restart sshd

回到控制台，将防火墙的22端口禁掉

至此，修改22端口完成。

图方便就在腾讯云平台买域名了，使用域名可以添加免费SSL证书，提高安全性，隐藏真实IP，直接使用域名访问酒馆更方便。

域名找一些便宜的顶级域就行，什么.fun什么的，随便找一个便宜的就行，大概就十几块一年。

具体配置腾讯云站内有详细教程，按教程来就行。

详细步骤（简易搭建）

  1. 拉取酒馆：

    1. 切换路径cd /usr/local/games/

    2. sudo git clone https://github.com/SillyTavern/SillyTavern -b release

    3. 输出如下图：

  2. 进入SillyTavern目录执行start脚本安装相关依赖（建议赋权sudo chmod -R 777 SillyTavern/*），如图：
    输出如图：

  3. 修改配置文件sudo vim config.yaml

    1. listen:ture

    2. port:7890(自定义，需确保防火墙释放端口，分别是腾讯控制台防火墙和终端使用ufw放行，具体操作可以看[修改22端口](#### 修改22端口))

    3. basicAuthMode:true（登录需要账号密码）

    4. basicAuthUser(设置登录账号密码)

    5. whitelistMode:false（禁用白名单，不限制ip）

  4. 再次执行start脚本启动，输入ip+端口进行访问，输入账号密码进入，浏览器页面如图：

  5. 完结撒花！！

进阶操作（caddy反代–推荐）

验证安装caddy version

caddy是一个轻量级的反代服务器，并且可以自动配置管理SSL证书。

简易配置文件CaddyFile如下：

使用sudo systemctl start caddy 启动caddy，使用sudo systemctl status caddy查看状态，使用浏览器输入域名进入sillytavern页面，至此caddy代理域名访问完成。

后台启动sillytavern（pm2实现）

在 Ubuntu 上安装 PM2（Node.js 进程管理器）可以为你的应用提供 进程守护、负载均衡、日志管理 等功能。

如果只是单纯的使用脚本启动，ssh连接断开后sillytavern就会自动断连，使用pm2可以方便后台启动以及日志管理

步骤如下

  1. 全局安装sudo npm install pm2@latest -g

  2. 验证pm2 --version

  3. 进入sillytavern目录执行pm2 start start.sh --name sillytavern即可

  4. 可使用pm2 list、pm2 monit、pm2 start logs sillytavern进行进程监控以及日志查看
